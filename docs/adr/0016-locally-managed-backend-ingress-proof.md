# Locally Managed Backend Ingress Proof

Lantern's first backend ingress slice will expose the self-hosted backend through a Cloudflare Tunnel on the temporary hostname `lantern-api.royzhao.dev`, with `cloudflared` running on the Ubuntu host, loopback `nginx` as the local origin boundary, and the tunnel itself intended to survive into later same-origin `/api/*` integration while the proof hostname is removed after cutover. We chose this over Terraform-managed tunnel provisioning or remote-config-first tunnel management because the documented locally managed Cloudflare Tunnel flow is cleaner for initial enrollment and host credential handling, keeps the source of truth for origin routing on the machine that actually serves traffic, avoids introducing secret material or awkward tunnel enrollment workflows into Terraform state, and still gives us a realistic end-to-end ingress path to validate before we change the live frontend edge contract.
