# Locally Managed Backend Ingress Proof

Lantern's first backend ingress slice will expose the self-hosted backend through a Cloudflare Tunnel on the initial proof hostname `lantern-api.royzhao.dev`, with `cloudflared` running on the Ubuntu host, loopback `nginx` as the local origin boundary, and the tunnel itself intended to survive into later same-origin `/api/*` integration. We chose this over Terraform-managed tunnel provisioning or remote-config-first tunnel management because the documented locally managed Cloudflare Tunnel flow is cleaner for initial enrollment and host credential handling, keeps the source of truth for origin routing on the machine that actually serves traffic, avoids introducing secret material or awkward tunnel enrollment workflows into Terraform state, and still gives us a realistic end-to-end ingress path to validate before we change the live frontend edge contract.

ADR 0017 later promotes `lantern-api.royzhao.dev` from a public proof hostname to a Cloudflare Access-protected backend origin for CloudFront.
