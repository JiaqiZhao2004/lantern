# Self-Hosted Backend Runtime And Durability

Lantern's first production backend slice will run on a single Ubuntu LTS personal server as a self-hosted primary, with Docker Compose as the runtime boundary, `nginx` as the local reverse proxy, Cloudflare Tunnel as the only public ingress path, and Postgres on the same machine with cloud-backed logical backups to S3 for durability. We chose this over managed runtime or database infrastructure for the first phase because it keeps cost and operational sprawl down for a personal project while still enforcing a deliberate runtime model: versioned images from GHCR, explicit deployment scripts, manual-but-defined Alembic migration workflow, expand-contract-friendly schema evolution, minimal monitoring and email alerting, and clear documentation of deliberate limitations such as mounted Firebase admin credentials, narrow long-lived backup credentials, and deferred deeper host hardening or secrets-management redesign.
